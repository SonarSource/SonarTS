If a local variable, function or class, a private method or an import is declared but not used, it is dead code and should be removed. Doing so will improve maintainability because developers will not wonder what the variable or function is used for.

Noncompliant Code Example

import { SomethingUnused } from "myModule"; // Noncompliant, SomethingUnused is never used

function numberOfMinutes(hours: number) {
  var seconds = 0;   // Noncompliant, seconds is never used
  return hours * 60;
}

Compliant Solution

function numberOfMinutes(hours: number) {
  return hours * 60;
}

Exceptions

All names starting with _ are ignored in this rule.
